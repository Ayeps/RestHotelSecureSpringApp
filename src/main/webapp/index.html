Secure RESTful Web Service API

Go to Hotels App

REST is an acronym for Representational State Transfer (see: http://en.wikipedia.org/wiki/REST. This means that all interaction from client to server is via normal HTTP methods: GET, POST, PUT, PATCH, DELETE and URIs (Universal Resource Identifiers) that represent the data access methodology. All data payloads to and from the server are in JSON format and are secured via SSL (in this example we are using a self-signed cert, hence the "-k" curl option). In the API details below, replace "localhost:8181" with the actual server and port used. Usually this will be 443 in production, and if that's the defauilt, which it usually is, you don't need to use the port number.

Retrieve All Hotel Records
HTTP Method (SSL Required): GET
URI: https://localhost:8181/RestHotelSecureSpringApp/api/v1/hotels
Response Data Type: application/json

Create New Hotel Record
HTTP Method (SSL Required): POST
URI: https://localhost:8181/RestHotelSecureSpringApp/api/v1/hotels
Request Payload Content Type: application/json 
     jgl-mac: curl -i -k -X POST -H "Content-Type:application/json" -d '{  "address" : "1234 Street",  "city" : "Chicago", "name" : "Best Western", "zip" : "60612" }' https://localhost:8181/RestHotelSecureSpringApp/api/v1/hotels

        HTTP/1.1 201 Created
        Server: Apache-Coyote/1.1
        Location: https://localhost:8181/RestHotelSecureSpringApp/api/v1/hotels/12

Update Hotel Record By ID
